# Touchpoint — alignment check

A gut-check on whether the user's time is going where their goals say it should. **Advisory** — this is a
conversation that may nudge `home.md`, not a mass edit, and it writes no dedicated note.

## Steps

1. **Gather intent** — read `goals/`, active `projects/`, and the priorities/open-goals section of
   `home.md`.
2. **Gather activity** — read recent `plans/`, `sessions/`, `meetings/`, and `devlog/` over an
   explicit absolute date window (default ~the last 1–2 weeks).
3. **Assess alignment** — for each goal/priority: is recent time actually moving it forward? What's
   **drifting or neglected**? What's eating time but tied to no stated goal? Note any dependencies or
   bottlenecks blocking the high-priority work.
4. **Discuss with the user** — present the read concisely; a `goal → recent activity → verdict` table
   works well. Ask probing questions where his intent or priorities are unclear. This is a dialogue,
   not an edit pass.
5. **Recommend priorities** — concrete next focus, what to drop or defer, what to escalate.
6. **Optionally refresh `home.md`** — only if the user agrees, update its priorities / recent-decisions.
   No dedicated touchpoint note.
7. **Append to `log.md`**: a one-line `## [YYYY-MM-DD] touchpoint | <scope>` noting the check ran and
   the key recommendation.

## Done when

- Recent activity is assessed against goals with citations, clear priority recommendations are given,
  `home.md` is optionally refreshed, and the check is logged.
